SECTION 1 - PROJECT OVERVIEW
================================================================================

Animotion is a gamified animation learning platform designed to teach animation
through doing, not watching. Users learn the 12 principles of animation and
specialised skills across 2D, 3D, VFX, Cinematography, and Motion Graphics,
progressing through an XP-based game system while building real animations in
a built-in editor.

PURPOSE:
Make animation education accessible, engaging, and rewarding. Most animation
courses are passive (watch tutorials). Animotion forces active participation:
complete interactive lessons, submit animations for AI feedback, build projects
in the editor, and compete on the leaderboard.

WHAT MAKES IT UNIQUE:
- Combines gamified learning (Duolingo-style) with a real creative tool
  (animation editor)
- AI coach that tracks 12 individual skills and gives structured, personalised
  feedback
- Every feature is interconnected: lessons improve skills, skills unlock
  recommendations, editor exports feed the AI coach
- Completely free platform

SECTION 2 - CORE FEATURES
================================================================================

------------------------------------------------------------
GAMIFIED LEARNING SYSTEM
------------------------------------------------------------

XP is earned from: completing lessons, passing quizzes, submitting tasks,
watching videos, chatting with the AI coach, and saving projects.

Levels are calculated from XP using the formula:
  level = floor(sqrt(xp / 90))

9-TIER RANK SYSTEM (in order):
  1. Bronze        -       0 XP
  2. Silver        -     500 XP
  3. Gold          -   1,500 XP
  4. Platinum      -   4,000 XP
  5. Diamond       -   9,000 XP
  6. Master        -  18,000 XP
  7. Pro Animator  -  35,000 XP
  8. Grandmaster   -  60,000 XP
  9. God-like      - 100,000 XP (nearly impossible)

Daily Quests: 7 quests per day, each awarding XP on completion.
Level-up modal with animated celebration when crossing a level threshold.
Floating XP toasts appear at the mouse position when XP is earned.

------------------------------------------------------------
DUOLINGO-STYLE LESSON PROGRESSION
------------------------------------------------------------

Lessons are organised into Worlds > Sections > Lessons.
Worlds unlock sequentially (complete all lessons in World N to unlock N+1).
Within a world, sections and lessons unlock progressively.

Each lesson has 5 steps:
  1. Intro
  2. Content
  3. Quiz
  4. Task
  5. Done

Quizzes are multi-question arrays (3-5 questions). Score-based bonus XP
is awarded per quiz. Task requires minimum 5-character response before
submission. Completing a lesson dispatches skill deltas defined per-lesson
in the curriculum.

------------------------------------------------------------
2D AND 3D ANIMATION EDITOR
------------------------------------------------------------

Canvas-based frame-by-frame 2D animation (580x360px default).
Layers panel: add, rename (double-click), toggle visibility, delete layers.
Timeline: add/delete frames, mark keyframes, scrub via click.
Onion skin toggle.
ALT-based keybind system (fully customisable in editor settings panel).
Export modal: MP4 (via MediaRecorder) or GIF (via gif.js CDN), with
resolution, FPS, and quality settings plus a progress bar.
Project save/load via Supabase (animation_projects table).
Auto-save every 15 seconds to localStorage and Supabase.
Project settings panel: resolution, FPS, background colour, duration.

------------------------------------------------------------
AI COACH
------------------------------------------------------------

Powered by Claude (Anthropic API), proxied through /api/chat server-side route.
Persona: "Ani" - friendly, encouraging, expert in all 12 Disney animation
principles.
Returns structured skillUpdate JSON block in every response.
Parses skill deltas and dispatches GAIN_SKILLS to global state.
Displays 12 animated skill bars in the coach sidebar.
Smart suggestions: recommends specific lessons based on weakest skills.
Chat history maintained in local component state for session.

------------------------------------------------------------
VIDEO-BASED LESSONS
------------------------------------------------------------

All lesson content includes a YouTube embed via the YouTubeEmbed component.
Lazy-load: shows thumbnail with play button, plays inline on click.
Videos pulled from the curriculum's videoId field (extracted from full YouTube
URLs via the yt() helper function).
Video Library page (/videos) aggregates all curriculum videos by world/section
with category filter tabs.
Watching a video awards +20 XP.

------------------------------------------------------------
SKILL SYSTEM
------------------------------------------------------------

12 skills:
  - Timing and Spacing
  - Squash and Stretch
  - Anticipation
  - Follow Through
  - Weight and Physics
  - Smoothness
  - Creativity
  - Consistency
  - Cinematography
  - Motion Design
  - VFX Understanding
  - Storytelling

All skills start at 0 for every new user. No preloaded data.

Skills increase through:
  - AI coach conversations (parsed from skillUpdate JSON)
  - Lesson completions (per-lesson skills object in curriculum)
  - Quiz performance

Each skill has a colour and icon defined in the SKILLS array in
lib/curriculum.js.

Skills are persisted to Supabase user_profiles.skills (jsonb) and
localStorage.

Profile page shows all 12 skill bars with level labels:
  Novice > Beginner > Intermediate > Advanced > Master

Strongest and weakest skill highlights shown on profile page.

SECTION 7 - AI COACH SYSTEM
================================================================================

Main file:   app/coach/page.jsx
API Route:   app/api/chat/route.js (server-side, keeps API key secret)

------------------------------------------------------------
HOW IT WORKS
------------------------------------------------------------

Step 1: User sends a message via the chat input.
Step 2: Full conversation history sent to /api/chat via POST.
Step 3: Server calls Anthropic API with system prompt + conversation history.
Step 4: Response parsed for <skillUpdate> JSON tag.
Step 5: Skill deltas extracted and dispatched via GAIN_SKILLS.
Step 6: XP from skillUpdate.xp field dispatched via GAIN_XP.
Step 7: Cleaned response (tag removed) displayed in chat.

------------------------------------------------------------
SYSTEM PROMPT SUMMARY
------------------------------------------------------------

Persona: "Ani", friendly animation coach.
Expert in 12 Disney principles, timing, spacing, weight, 2D/3D animation,
VFX, acting, and cinematography.
Must return <skillUpdate> JSON after every response.
Only increase relevant skills (0-15 per skill per response).
XP field: 10-50 per response.
Keep replies to 2-4 short paragraphs.

------------------------------------------------------------
SKILL UPDATE FORMAT
------------------------------------------------------------

<skillUpdate>
{
  "timing": 0,
  "squash": 0,
  "anticipation": 0,
  "followThrough": 0,
  "weight": 0,
  "smoothness": 0,
  "creativity": 0,
  "consistency": 0,
  "cinematography": 0,
  "motionDesign": 0,
  "vfx": 0,
  "storytelling": 0,
  "xp": 20
}
</skillUpdate>

Only increase skills relevant to the conversation.
Use values 0-15 per skill per message.
XP should be 10-50 per message.

------------------------------------------------------------
UI COMPONENTS
------------------------------------------------------------

Chat interface with Mascot avatar ("Ani") showing mood states:
  - happy
  - thinking
  - excited
  - sad

Animated typing indicator (3 pulsing dots).
12 animated skill bars in sidebar panel with colour per skill and delta badge
on update.
Smart suggestions: top 3 weakest skills mapped to recommended lessons.
Upload zone for animation file (UI ready, analysis logic is placeholder).
Quick prompt buttons for common animation questions.

SECTION 8 - LEARNING STRUCTURE
================================================================================

Lessons are organised as: World > Section > Lesson

All lesson data lives in lib/curriculum.js. The WORLDS array has this structure
per world:

  {
    id,
    title,
    color,
    icon,
    unlockedByDefault,
    desc,
    sections: [
      {
        id,
        title,
        lessons: [
          {
            id,
            title,
            xp,
            time,
            diff,
            icon,
            desc,
            content,       // multi-paragraph text
            videoId,       // YouTube video ID (extracted via yt() helper)
            quiz: [        // array of 3-5 questions
              { q, opts: [], correct }
            ],
            task,          // written task prompt
            skills: {      // skill deltas awarded on completion
              timing: 10,
              squash: 5,
              // ...
            }
          }
        ]
      }
    ]
  }

------------------------------------------------------------
WORLDS AND CONTENT
------------------------------------------------------------

World 1: 2D Animation
  Sections: Fundamentals, Movement, Advanced, Bonus
  Content: 12 principles, timing, squash/stretch, bouncing ball, walk cycle,
  anticipation, keyframes, weight, acting, polishing, bonus lessons

World 2: 3D Animation
  Sections: Fundamentals, Movement, Advanced, Bonus
  Content: What is 3D, Blender basics, keyframes, graph editor, bouncing ball,
  camera animation, rigging, character animation, physics, bonus lessons

World 3: VFX
  Sections: Fundamentals, Movement and Effects, Advanced, Bonus
  Content: What is VFX, VFX vs CGI, After Effects basics, green screen, motion
  tracking, text animation, explosions, lightning, compositing, bonus lessons

World 4: Cinematography
  Sections: Fundamentals, Camera Techniques, Advanced, Bonus
  Content: What is cinematography, shot types, rule of thirds, lighting, camera
  movement, depth of field, film blocking, colour theory, cinematic
  composition, bonus lessons

World 5: Motion Graphics
  Sections: Fundamentals, Text and UI Animation, Advanced, Bonus
  Content: What is motion graphics, After Effects, keyframes/easing, motion
  design principles, text animation, logo animation, UI animation, transitions,
  kinetic typography, looping, bonus lessons

------------------------------------------------------------
ALL_LESSONS FLAT ARRAY
------------------------------------------------------------

ALL_LESSONS is exported from lib/curriculum.js as:

  export const ALL_LESSONS = WORLDS.flatMap(w =>
    w.sections.flatMap(s =>
      s.lessons.map(l => ({
        ...l,
        worldId: w.id,
        worldTitle: w.title,
        worldColor: w.color,
        worldIcon: w.icon,
        sectionId: s.id,
        sectionTitle: s.title
      }))
    )
  );

This flat array is used everywhere lessons need to be listed, searched,
or referenced without knowing which world or section they belong to.

SECTION 10 - PROJECT GOAL
================================================================================

Animotion should feel like a mix of:

  DUOLINGO  - structured, gamified learning path with XP, quests, and
              level-ups. Every session has a clear goal and reward.

  BLENDER   - a real creative tool where you make actual animations.
              Not a sandbox toy, but a production-capable editor.

  AI MENTOR - personalised, conversational feedback that grows with you.
              Ani knows your weaknesses and helps you improve them.

  A GAME    - every action is rewarded, every skill improves, every session
              leaves you better than before.

The platform should never feel like homework. It should feel like levelling up
a character. The user's animator skills are their character stats. The lessons
are quests. The editor is their crafting table. Ani is their guide.

SECTION 11 - IMPORTANT RULES FOR FUTURE DEVELOPMENT
================================================================================

------------------------------------------------------------
PRODUCT RULES
------------------------------------------------------------

Rule 1: No fake data in production features.
  The leaderboard must only show real users. Skills must reflect real progress.
  XP must be earned. Never use hardcoded arrays of fake users or stats.

Rule 2: Fresh start for every new user.
  Never preload XP, levels, skills, or completed lessons. Every account starts
  at zero and earns everything through the platform.

Rule 3: Every feature must feel interactive.
  No static screens, no dead ends. There should always be something to click,
  earn, or discover.

Rule 4: Gamification is non-negotiable.
  Every completed action should feel rewarding. XP toast, animation, or audio
  feedback on every meaningful action.

Rule 5: Learning by doing.
  Lessons must have quizzes and practical tasks, not just reading material.
  The user must produce something in every lesson.

------------------------------------------------------------
TECHNICAL RULES
------------------------------------------------------------

Rule 6: w.lessons does not exist.
  Worlds have SECTIONS. Sections have LESSONS. Always use:
    w.sections.flatMap(s => s.lessons)
  Or use the ALL_LESSONS flat array from lib/curriculum.js.
  Never reference w.lessons directly. This will cause a runtime crash.

Rule 7: Skills always start at 0.
  Use DEFAULT_SKILLS from lib/curriculum.js. This object is merged safely in
  the reducer. Never hardcode skill values for new users.

Rule 8: GAIN_SKILLS not UPDATE_SKILLS.
  The store action type is GAIN_SKILLS with a deltas object. Example:
    dispatch({ type: 'GAIN_SKILLS', deltas: { timing: 10, squash: 5 } })

Rule 9: Server-side secrets only.
  ANTHROPIC_API_KEY must never have the NEXT_PUBLIC_ prefix. It is always
  proxied through /api/chat and never exposed to the browser.

Rule 10: Sanitize all inputs.
  Use sanitize() from lib/utils.js before storing any user-generated text.
  This prevents XSS and injection attacks.

Rule 11: All pages need 'use client'.
  Every page component must start with 'use client'. Exceptions are:
    - app/page.jsx (server-side redirect only)
    - All route.js files (API routes, server-side only)

Rule 12: DB operations must validate ownership.
  All Supabase writes must include .eq('user_id', userId) or .eq('id', userId)
  to prevent cross-user data access. Row Level Security is also enforced at
  the database level.

------------------------------------------------------------
DESIGN RULES
------------------------------------------------------------

Rule 13: Keep it Wii/Nintendo.
  Soft colours, rounded corners, playful interactions. No harsh shadows,
  no aggressive colours, no corporate feel.

Rule 14: Think mobile-friendly.
  Layouts should degrade gracefully. Never assume a wide screen.

Rule 15: Footer on every page.
  "Made by Posh :)" must appear fixed bottom-right on all authenticated pages.
  It is in components/layout/Footer.jsx and rendered by AppShell.

Rule 16: Floating feedback button always visible.
  The 💬 button is rendered in AppShell.jsx at the bottom-right of the screen
  on every authenticated page. It opens FeedbackModal which shows the
  contact email: takuwhy@gmail.com

------------------------------------------------------------
ROW LEVEL SECURITY
------------------------------------------------------------

RLS is enabled on both tables.
Users can only read and write their own rows.
Leaderboard query uses a separate policy allowing authenticated users to read
display_name, xp, and level from all profiles.

------------------------------------------------------------
TRIGGERS
------------------------------------------------------------

handle_new_user():
  Fires AFTER INSERT on auth.users.
  Automatically creates a clean user_profiles row with:
    xp = 0, level = 1, completed_lessons = {}, has_seen_tutorial = false,
    skills = {}
  Uses ON CONFLICT (id) DO NOTHING so it is safe to run multiple times.

set_updated_at():
  Fires BEFORE UPDATE on both tables.
  Automatically sets updated_at = now() on every update.

------------------------------------------------------------
AFTER RUNNING schema.sql
------------------------------------------------------------

1. Go to Auth > Providers > Google and enable Google OAuth.
   Add your Google Client ID and Client Secret.

2. Go to Auth > URL Configuration > Site URL and set your domain.
   Local: http://localhost:3000
   Production: https://your-app.vercel.app

3. Go to Auth > URL Configuration > Redirect URLs and add:
   https://your-app.vercel.app/auth/callback
